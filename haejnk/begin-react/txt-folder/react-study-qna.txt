useMemo 함수의 두번째 파라미터의 역할?
내부적으로 어떻게 돌아가나

일단 이 함수는 데이터 단위로 사용된다. 
(이 함수를 기반으로 만들어진 useCallback 함수는
함수단위로 사용됨.)
페이지가 렌더링 될 때 변화한 값이 있을 경우 
메모리의 새 공간을 소모한다.
이렇게 되면 메모리를 낭비하게 된다.
함수를 사용하여 메모리를 재사용하게 된다.
그런데 두번째 파라미터에 값이 없거나 빈 배열을 넣을 경우
함수를 사용하여 렌더링 되는 값은 변화가 없다.
(예를 들어 값이 증가되게 하고 싶은데, 두번째 파라미터로 
설정해 주지 않을 경우, 해당 값은 상태 변화가 없게 됨)

메모리를 재사용 하면서 값은 변경하고 싶을 때
두번째 파라미터에 해당 값을 설정해 준다.
